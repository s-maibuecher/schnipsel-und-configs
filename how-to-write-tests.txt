Ich werde das Functional Testing wie im Buch "Test-driven Delopment with Pyhton" von Harry Percival aufziehen, Kapitel:
"Unit Tests, and How They Differ from Functional Tests".
Das bedeutet, dass das Functional Testing die Code Spezifikation ersetzt. Man schreibt die User-Story hier nieder und
definiert die passenden Tests dazu, die zunächst failen (expected fail) und nach der korrekten Implementierung
ohne Fehler durchlaufen.
